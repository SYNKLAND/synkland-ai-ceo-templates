# Why Savage Mode — 設計思想

> 公開タイミング：V2-F α 版リリース直前。MVP の実体に紐付けて公開することで模倣者を「ただの口が悪いフォロワー」にとどめる。

---

## なぜ「毒舌」が必要か

OpenAI / Anthropic / Microsoft の汎用 LLM は、RLHF（人間のフィードバックによる強化学習）によって以下の出力傾向に強制されている：

1. **婉曲表現の優先** — 「〜と思われます」「〜のように見受けられます」
2. **複数選択肢の併記** — 「Option A も B も C も検討に値します」
3. **意思決定の保留** — 「最終判断はお客様自身で」
4. **失敗の指摘回避** — 「成長の機会と捉えることもできます」

これは大企業の社内利用には合理的だが、**ひとり社長には致命的**である。

## ひとり社長が必要としている AI

ひとり社長は本業 + 経理 + 営業 + 採用 + 法務 + 戦略 を 1 人で回している。**意思決定の速度が事業の生死を決める**。彼らに必要なのは：

- 「**死に金**」を死に金と呼ぶ AI
- 来月の意思決定を **1 つに絞る** AI
- 「**3 ヶ月で潰れる**」と言える AI
- 婉曲表現で時間を奪わない AI

## なぜ税理士・弁護士・コンサルでは不可能か

| 職業 | 制約 |
|---|---|
| 税理士 | 税法の助言は可能だが、経営判断への踏み込みは越境 |
| 弁護士 | 法的リスク以外の踏み込みは契約外 |
| 中小企業診断士 | 一般論レベルの提言で、踏み込んだ批判は契約解除リスク |
| 経営コンサル | 月額 ¥300k+ の費用感 |
| ChatGPT | RLHF で去勢されている |

→ **個人開発者だけが、ブランドリスクを取って真実を提供できる構造**

## 模倣されたらどうするか

このプロンプトを GitHub に OSS で公開する理由は **3 つ**：

1. **思想の先行宣言**: 「なぜこの AI は厳しいのか」を文書化することで、模倣者は思想なき劣化コピーになる
2. **発言責任のブランド化**: 公開された思想を背負っているのは SYNKLAND という固有名詞
3. **コミュニティ駆動の改善**: PR で他の Indie 開発者がプロンプトを鍛える

模倣の本当の defense は「**プロンプトを隠すこと**」ではなく「**プロンプトと一緒に思想とブランドを背負うこと**」である。

## 限界（誠実さのために明記）

- 毒舌モードは万人に合うものではない。**丁寧モード**を切替で用意している
- 「踏み込み」は時に誤りうる。最終判断は人間
- これは経営助言ではなく**データ分析**である（法的免責）
- ユーザーが感情的に傷つく可能性がある — それを承知で使う設計
